What is Kryon?

Kryon is a UI cartridge system for native applications. Write interface structure in simple .kry files, compile it into readable C, and keep the runtime small enough for desktop, mobile, web, and embedded targets.

The Kryon IDE is built around the same source model: open a project or file path, edit .kry directly, see a live preview, and click a widget to jump back to the source line that generated it.

Core Features

Widget-Focused .kry

Pages and reusable widgets stay in concise source files with explicit typed variables and direct compiler errors for unknown syntax.

Clean C Output

The compiler emits normal C and headers in a fresh generated folder for direct interop.

Live Preview

Hot reload rebuilds real .kry output, keeps the last good state during compilation, and shows diagnostics when the build fails.

Source-Aware Inspection

Preview widgets carry source metadata, so inspection clicks select the generated UI element and open the exact .kry line.

Philosophy

  • Everything visible is a widget - source, bounds, state, and inspection all belong to the widget tree.
  • Source stays primary - .kry files own screens and reusable UI pieces; C is the generated interop target.
  • The IDE is source-aware - file browser, source editor, live preview, inspector, diagnostics, and search stay tied to .kry files.
  • Default private API - only exported functions enter generated headers, keeping build boundaries small.
  • Native first - direct C, raylib-style calls, and predictable runtime behavior across targets.

Get Started

Build the runtime and tools:

git clone https://github.com/kryonlabs/kryon.git
cd kryon
make
make install

A simple .kry shape:

cimport "kryon.h"

pub fn AppHome(app: InbeApp*) {
    var y: int = ScaleUIPx(24)

    draw DrawUIText("Inner Breeze", ScaleUIPx(20), y, UI_TEXT_24, GetThemeText())
    draw app_draw_bottom_nav(app)
}

Open a project or a single .kry file in the IDE:

kryon .
kryon path/to/app.kry
