What is Kryon?

Kryon is a UI cartridge system for native applications. Write interface structure in simple .kry files, compile it into readable C, and keep the runtime small enough for desktop, mobile, web, and embedded targets.

The IDE is part of the system: every visible widget can carry source metadata, so preview clicks can jump back to the line that generated the widget.

Core Features

Widget-Focused .kry

Pages and reusable widgets stay in concise source files instead of hand-written screen C.

Clean C Output

The compiler emits normal C and headers in a fresh generated folder for direct interop.

Live Widget Preview

Hot reload keeps the preview tied to current .kry source instead of a fake mock view.

Widget Gallery

Search registered widgets, drop them into pages, move them visually, and keep source links intact.

Philosophy

  • Everything visible is a widget - source, bounds, state, and inspection all belong to the widget tree.
  • Source stays primary - .kry files own screens and reusable UI pieces; C is the generated interop target.
  • The IDE is modular - file browser, source editor, preview, inspector, widget gallery, and search are swappable panes.
  • Default private API - only exported functions enter generated headers, keeping build boundaries small.
  • Native first - direct C, raylib-style calls, and predictable runtime behavior across targets.

Get Started

Build the runtime and tools:

git clone https://github.com/kryonlabs/kryon
cd kryon
make

A simple .kry shape:

include "kryon.h"

pub fn AppHome(app: *InbeApp) {
    var y: int = ScaleUIPx(24)

    draw DrawUIText("Inner Breeze", ScaleUIPx(20), y, UI_TEXT_24, GetThemeText())
    draw app_draw_bottom_nav(app)
}

Compile and run through the IDE:

make run PROJECT=../inbe
